# EP4N SEO Backend Technical Specification

**Version:** 2.0 (Production Ready - Multi-language Ready)

## Objective

Build a scalable, multilingual SEO architecture following Google Search
and Schema.org best practices.

# 1. URL Structure

Use language prefixes.

``` text
/ar/steam/steam-wallet-usa/steam-wallet-gift-card-10-usd
/en/steam/steam-wallet-usa/steam-wallet-gift-card-10-usd
```

Rules: - One Primary URL per language. - Product ID/UUID is the
permanent identifier. - URLs generated by Routing only. - Unique Slugs
within each language.

# 2. Canonical URL

-   Generate dynamically.
-   Canonical points to the current language version only.
-   Never canonical Arabic to English or vice versa.

# 3. Hreflang

Generate alternate links for every translated page.

``` html
<link rel="alternate" hreflang="ar" href="https://www.ep4n.info/ar/...">
<link rel="alternate" hreflang="en" href="https://www.ep4n.info/en/...">
<link rel="alternate" hreflang="x-default" href="https://www.ep4n.info/">
```

# 4. Breadcrumb Navigation

Generate dynamically per language. Visible HTML must match Schema.

# 5. BreadcrumbList Schema

Generate JSON-LD from the same source as HTML.

# 6. Product Schema

Generate dynamically. Translate only localized fields: - name -
description - category Keep SKU, price, availability identical where
applicable. Do not output optional properties without real data.

# 7. Organization Schema

Generate globally once.

# 8. Website Schema

Generate globally. Include SearchAction only if internal search exists.

# 9. Dynamic SEO Meta

Generate per language: - Title - Description - OpenGraph - Twitter
Cards - Canonical

# 10. XML Sitemap

Use Sitemap Index.

``` text
/sitemap.xml
/sitemap-ar.xml
/sitemap-en.xml
```

Rules: - Separate sitemap for each language. - No parameter URLs.

# 11. Internal Linking

Link related products, variants and category pages within the current
language.

# 12. Product Variants

Each variant has its own page. Cross-link variants within the same
language.

# 13. URL Management & Redirect Strategy

-   Routing-generated URLs only.
-   Slug History.
-   Never reuse Slugs.
-   Automatic 301 Redirect on slug change.
-   Update Canonical and Sitemap automatically.
-   Temporarily unavailable =\> OutOfStock.
-   Discontinued with replacement =\> 301.
-   Discontinued without replacement =\> 410 Gone.

## URL Parameters

Never include parameter URLs in sitemap. Canonical to clean URL when
appropriate. Use noindex,follow where required.

# 14. Translation Architecture

Each product supports: - Product ID - Language - Localized Slug -
Localized Title - Localized Description - Localized Meta Title -
Localized Meta Description - Localized Breadcrumb

Changing one language must not affect URLs of other languages.

# 15. Testing & Validation

Validate on Staging: - Canonical - Hreflang - Redirects - Schema - Rich
Results - Sitemap - Broken Links - Core Web Vitals - OpenGraph - Twitter
Cards
