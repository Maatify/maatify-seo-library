# EP4N SEO Backend Technical Specification

**Version:** 1.0 (Production Ready)

## Objective

Build a scalable, search-engine-friendly backend architecture following
current Google Search and Schema.org best practices while supporting
future growth without structural changes.

------------------------------------------------------------------------

# 1. URL Structure

Use a hierarchical URL structure that reflects the real product
hierarchy.

Example:

``` text
/steam/steam-wallet-usa/steam-wallet-gift-card-10-usd
/steam/steam-wallet-ksa/steam-wallet-gift-card-ksa-sar-50
/playstation/playstation-plus-12-month-usa
```

Requirements:

-   One Primary URL per product.
-   Unique Slugs.
-   Never generate multiple URLs for the same product.
-   URLs are generated by the Routing layer.
-   Never store complete URLs inside the database.
-   Product ID / UUID is the permanent identifier.

------------------------------------------------------------------------

# 2. Canonical URL

Generate Canonical dynamically from the Routing layer.

``` php
$productCanonical = buildProductUrl($product);

echo '<link rel="canonical" href="' .
htmlspecialchars($productCanonical, ENT_QUOTES, 'UTF-8') .
'" />';
```

Rules:

-   Always reference the Primary URL.
-   Update automatically when Slug changes.
-   Never use Canonical as a workaround for duplicate URLs.

------------------------------------------------------------------------

# 3. Breadcrumb Navigation

Generate Breadcrumb dynamically.

Example:

``` text
Home
>
Steam
>
Steam Wallet USA
>
Steam Wallet Gift Card 10 USD
```

Requirements:

-   All items clickable.
-   Generated from the same data source used for Schema.
-   Visible to users.

------------------------------------------------------------------------

# 4. BreadcrumbList Schema (JSON-LD)

Generate HTML Breadcrumb and JSON-LD from the same source (Single Source
of Truth).

Rules:

-   Same array builds HTML and Schema.
-   Include every breadcrumb item.
-   Keep `item` for all elements, including the current page.
-   Generate dynamically.

------------------------------------------------------------------------

# 5. Product Schema

Generate Product Schema dynamically.

Required:

-   name
-   description
-   image
-   sku
-   brand
-   category

Offers:

-   price
-   priceCurrency
-   availability
-   seller
-   itemCondition
-   url

Optional:

-   aggregateRating
-   review
-   gtin
-   mpn
-   priceValidUntil

Rules:

-   Never output optional fields unless real data exists.
-   Temporarily unavailable products must use `OutOfStock`.

------------------------------------------------------------------------

# 6. Organization Schema

Generate once globally.

Include:

-   Organization
-   Name
-   Website
-   Logo
-   SameAs

------------------------------------------------------------------------

# 7. Website Schema

Generate once globally.

Include:

-   Website
-   URL
-   Name
-   SearchAction

Only output SearchAction if a working internal search engine exists.

------------------------------------------------------------------------

# 8. Dynamic SEO Meta

Generate dynamically:

-   Meta Title
-   Meta Description
-   Canonical
-   Robots
-   OpenGraph
-   Twitter Cards

------------------------------------------------------------------------

# 9. XML Sitemap

Generate automatically.

Include:

-   Products
-   Categories
-   Static Pages

Include:

-   lastmod
-   priority
-   changefreq

Recommended:

-   Image Sitemap

Rules:

-   Never include parameterized URLs.
-   Update automatically after URL changes.

------------------------------------------------------------------------

# 10. Internal Linking

Every product page should link to:

-   Related products
-   Same game
-   Same store
-   Same region
-   Same currency
-   Same brand
-   Higher value products
-   Lower value products
-   Product variants

------------------------------------------------------------------------

# 11. Product Variants

Each variant has its own page.

Example:

``` text
Steam Wallet USA
- 10 USD
- 20 USD
- 50 USD
```

Rules:

-   Link all variants together.
-   ProductGroup Schema can be implemented later (Phase 2).

------------------------------------------------------------------------

# 12. URL Management & Redirect Strategy

## URL Stability

-   Routing generates URLs.
-   Never store full URLs in DB.
-   Product ID is permanent identity.

## Slug Policy

-   Unique Slugs.
-   Keep Slug History.
-   Never reuse old Slugs.

## Redirect Strategy

When a Slug changes:

-   Automatic 301 Redirect.
-   Update Canonical.
-   Update Sitemap.

## Product Lifecycle

### Temporarily unavailable

-   Keep page.
-   Product Schema = OutOfStock.
-   Display alternatives.

### Permanently discontinued with replacement

-   301 Redirect.

### Permanently discontinued without replacement

-   Return HTTP 410 Gone.

### Invalid URL

-   Return HTTP 404.

## URL Parameters & Faceted Navigation

Examples:

``` text
?page=2
?sort=price
?filter=usa
?brand=steam
```

Rules:

-   Never include parameter URLs in Sitemap.
-   Canonical points to clean URL unless intentionally indexed.
-   Use:

``` html
<meta name="robots" content="noindex,follow">
```

when appropriate instead of relying on robots.txt.

------------------------------------------------------------------------

# 13. Testing & Validation

Before Production Release:

-   Validate Canonical URLs.
-   Validate 301 Redirects.
-   Validate Slug History.
-   Validate Breadcrumb, Product, Organization and Website Schema.
-   Test with Google Rich Results Test.
-   Validate XML Sitemap.
-   Check Broken Links.
-   Validate Internal Links.
-   Validate Meta Tags.
-   Validate OpenGraph.
-   Validate Twitter Cards.
-   Validate Core Web Vitals.
-   Validate Image Optimization.

All testing must be completed on the Staging environment before
deployment.
